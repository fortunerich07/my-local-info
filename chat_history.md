# 📝 my-local-info 프로젝트 작업 및 대화 기록

이 문서는 **'my-local-info'** 웹 서비스 개발을 진행하며 나눈 주요 대화와 작업 처리 결과를 쉽게 요약 정리한 파일입니다.

---

## 1. 📂 이전 대화 기록 조회 방법
* **질문**: 이전 대화는 어디에서 불러올 수 있나요?
* **답변**:
  * **현재 대화방 이전 기록**: 대화창 화면을 위로 올리거나(스크롤) 마우스 휠을 굴려 확인 가능합니다.
  * **완전히 이전의 다른 대화방**: 개발 도구(VS Code 등) 대화창 주변의 **시계 모양(🕒)** 또는 **목록 모양(📝)의 'History(히스토리)'** 아이콘을 통해 과거 대화 세션을 다시 로드할 수 있습니다.

## 2. 💻 로컬 개발 서버 실행
* **상황**: 사이트 화면 테스트를 위해 개발 서버 작동 시도
* **문제 발생**: `npm run dev` 실행 시 윈도우 PowerShell의 **스크립트 실행 차단 보안 정책**에 의해 차단됨.
* **해결**: 보안 제한을 우회하기 위해 Windows CMD 배치 파일 호출 형태인 **`npm.cmd run dev`** 명령어로 우회하여 로컬 개발 서버를 정상적으로 켰습니다. (주소: `http://localhost:3000`)

---

## 3. 🐙 GitHub 소스 코드 업로드
* **요청**: 전체 소스 코드를 GitHub에 업로드 (민감 정보 배제 포함)
* **진행 및 문제 해결**:
  1. **보안 점검**: `.gitignore` 파일을 사전 분석하여 `.env` 및 민감한 비밀 키들이 깃허브에 노출되지 않도록 안전 설정을 확인했습니다.
  2. **환경 세팅**: 컴퓨터에 `git` 프로그램이 없음을 발견하고, 윈도우 자동 설치 도구인 `winget`을 이용해 **Git**과 **GitHub CLI**를 원클릭 설치했습니다.
  3. **깃 초기화**: `git init` 및 `git add` 후 첫 번째 커밋(`Initial commit`)을 완료했습니다.
  4. **계정 인증**: 브라우저 일회용 코드 인증(Device Flow)을 통해 `fortunerich07` 깃허브 계정으로 정상 로그인했습니다.
  5. **저장소 업로드**: GitHub에 새 저장소(`my-local-info`)를 생성한 후 코드를 최종 푸시(Push) 완료했습니다.
  * **소스 코드 링크**: [https://github.com/fortunerich07/my-local-info](https://github.com/fortunerich07/my-local-info)

## 4. ⛅ Cloudflare Pages 웹 사이트 배포
* **요청**: 사이트 빌드 및 Cloudflare Pages 배포 서비스에 전송
* **진행 및 문제 해결**:
  1. **사이트 빌드**: `npm run build`를 구동하여 최적화된 정적 HTML/CSS 파일 빌드(`out` 폴더 생성)를 성공적으로 완료했습니다.
  2. **클라우드플레어 로그인**: `wrangler login`을 통해 사용자 브라우저 권한을 획득했습니다.
  3. **이메일 인증 에러**: 계정 이메일 인증이 완료되지 않아 프로젝트 생성이 막혔으나, 가입 이메일(`fortunerich07@gmail.com`)에서 인증 완료 처리를 해주셔서 해결되었습니다.
  4. **최종 배포**: 205개의 최적화된 소스 파일을 성공적으로 클라우드 서버에 전송하여 사이트를 공개했습니다.
  * **나의 라이브 웹사이트 주소**: [https://my-local-info-4u5.pages.dev](https://my-local-info-4u5.pages.dev)

## 5. 📝 2회차 작업 및 대화 기록 (블로그 고도화, 자동화 및 광고 연동)
* **상황**: 정적 블로그 기능 구축, 매일 자동으로 글을 쓰는 크롤러/AI 시스템 구축, 광고 수익화 및 기술적 SEO(검색 노출 최적화) 완료.
* **진행 및 문제 해결**:
  1. **정적 블로그 구축**:
     * `gray-matter`, `react-markdown` 라이브러리를 통해 로컬의 마크다운 글(`.md`)을 예쁜 블로그 본문으로 렌더링하는 페이지를 신설했습니다.
     * 메인 화면의 카드 클릭 시 상세 페이지 대신 블로그(`blog/[slug]`)로 즉시 넘어가도록 연결을 재조정했습니다.
  2. **매일 스스로 작동하는 AI 자동화 스크립트**:
     * `fetch-public-data.js`: 매일 공공데이터포털 API에서 성남/경기도의 새로운 공공 혜택 정보를 수집하고, 중복을 검사한 후 Gemini AI로 가공해 데이터를 갱신합니다.
     * `generate-blog-post.js`: 갱신된 데이터를 바탕으로 Gemini AI가 한글 블로그 원고(800자 이상, 3가지 추천 이유 및 신청 링크 포함)를 영문 키워드 파일명으로 자동 생성해 줍니다.
  3. **GitHub Actions 깃허브 자동 배포 스케줄러**:
     * 매일 아침 7시(KST) 및 코드가 올라갈(Push) 때마다 자동으로 웹사이트를 빌드하고 Cloudflare Pages로 쏘아 올리는 규칙 파일(`.github/workflows/deploy.yml`)을 생성했습니다.
     * **오류 해결**: 깃허브 토큰의 수정 권한 부족 에러는 `gh auth refresh` 코드로 재인증했고, 배포 도구(Wrangler v4)가 최소 사양으로 요구하는 **Node.js 22** 버전을 액션에 지정해 빌드 실패 오류를 말끔히 해결했습니다.
  4. **광고 수익화 연동**:
     * Google AdSense 및 쿠팡 파트너스 위젯 배너를 표시할 수 있는 `AdBanner.tsx` 및 `CoupangBanner.tsx` 컴포넌트를 설계했습니다.
     * 환경변수가 활성화된 상태에서만 광고가 안전하게 렌더링되며, 본문 하단 및 메인 페이지에 배치했습니다.
  5. **구글 검색 엔진 최적화 (SEO) 및 E-E-A-T 가이드라인 준수**:
     * **지도의 자동 제출**: Next.js 정적 익스포트 빌드 시 사이트맵(`sitemap.xml`)과 로봇 지침(`robots.txt`)이 오류 없이 생성되도록 dynamic 설정을 연동했습니다.
     * **기본 메타데이터 보강**: 소셜 카드 공유 포맷(OpenGraph)을 메인과 동적 상세 페이지에 모두 삽입했습니다.
     * **소개 페이지 신설**: 사이트 취지와 AI 및 공공데이터 활용을 밝히는 About 페이지(`/about`)를 추가하여 구글이 권장하는 콘텐츠 신뢰도 요건을 완비했습니다.
     * **AI 고지 & 원문 연동**: 모든 AI 작성글 하단에 로봇 안내 배너 및 공식 상세 웹주소로 바로가기 링크를 제공하여 E-E-A-T 신뢰성을 충족시켰습니다.
